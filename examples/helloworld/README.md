# HelloWorld CCPP Example

A minimal two-scheme example demonstrating the complete xdsl-ccpp compilation
and Fortran build flow.  All files — metadata, scheme implementations, generated
caps, and the build system — live in this directory.

---

## Directory contents

### Metadata and suite definition (inputs to xdsl-ccpp)

| File | Description |
|------|-------------|
| `hello_world_suite.xml` | Suite definition (XML frontend) |
| `hello_scheme.meta` | Scheme metadata for `hello_scheme` |
| `temp_adjust.meta` | Scheme metadata for `temp_adjust` |
| `hello_world_mod.meta` | Host model module metadata |
| `hello_world_host.meta` | Host model subroutine metadata |

### Fortran scheme and host implementations

| File | Description |
|------|-------------|
| `hello_scheme.F90` | `hello_scheme` physics implementation |
| `temp_adjust.F90` | `temp_adjust` physics implementation |
| `hello_world_mod.F90` | Host model data module (variables + verification) |
| `hello_world_host.F90` | Host model driver and main program |

### Generated by xdsl-ccpp (see Step 1 below)

| File | Description |
|------|-------------|
| `ccpp_kinds.F90` | Kind parameter definitions (`kind_phys` via ISO_FORTRAN_ENV) |
| `hello_world_suite_cap.F90` | Suite cap: lifecycle subroutines for `hello_world_suite` |
| `HelloWorld_ccpp_cap.F90` | Host-facing cap: `HelloWorld_ccpp_physics_run` etc. |

---

## Step 1 — Generate the Fortran caps with xdsl-ccpp

Run from the repository root on your development machine (requires Python/xdsl-ccpp):

```bash
ccpp_xdsl \
  --suites   examples/helloworld/hello_world_suite.xml \
  --scheme-files \
      examples/helloworld/hello_scheme.meta,\
examples/helloworld/temp_adjust.meta \
  --host-files \
      examples/helloworld/hello_world_host.meta,\
examples/helloworld/hello_world_mod.meta \
  -o examples/helloworld/
```

This writes `ccpp_kinds.F90`, `hello_world_suite_cap.F90`, and
`HelloWorld_ccpp_cap.F90` directly into `examples/helloworld/`.

---

## Step 2 — Compile and run inside the Docker container

A `gcc:latest` Docker container provides gfortran without requiring a
system-level Fortran compiler on the development machine.

Start the container with the repository mounted:

```bash
docker run --rm -it \
  -v /path/to/xdsl-ccpp:/work \
  -w /work \
  gcc:latest \
  bash
```

Then inside the container, build and run:

```bash
# Build
make -f examples/helloworld/Makefile

# Run the executable directly
make -f examples/helloworld/Makefile run

# Build and verify correctness (prints TEST PASSED or TEST FAILED)
make -f examples/helloworld/Makefile test

# Clean build artifacts
make -f examples/helloworld/Makefile clean
```

Build artifacts are written to `examples/helloworld/build/`.

## Expected output

```
 Answers are correct!
TEST PASSED
```

The `compare_temp()` function in `hello_world_mod.F90` verifies that after one
physics call:

- `hello_scheme_run` computed each `temp_midpoints` layer as the average of its
  two bounding `temp_interfaces` interface values.
- `temp_adjust_run` added 1.0 K to each layer.

## Python frontend alternative

The same suite can be defined using the Python API instead of XML:

```bash
ccpp_xdsl --py examples/helloworld/helloworld_py.py \
  -o examples/helloworld/
```
